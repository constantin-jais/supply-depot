# Roadmap

This is a contribution map, not a startup roadmap or a delivery promise. It shows where help is useful while keeping scope explicit.

## Now

- make dogfooding evidence visible through commands, fixtures, CI checks, generated reports, or linked docs;
- stabilize artifact contract fixtures;
- document storage/cache policy assumptions;
- improve provenance examples;
- keep CI, contracts, and security checks green.

## Next

- add example artifact manifests and reports;
- improve fail-closed policy errors;
- add contract tests for retention and verification;
- prepare the first alpha-quality depot contract.

## Later

- broader registry and cache integrations;
- release provenance across Gear components;
- hosted depot usage only when trust, tenancy, and retention boundaries are explicit.
